# Tools v2.0 需求文档

## 1. 项目目标
本项目的目标是提供一个可在统一平台内使用的 DevOps 工具平台，并将其打包为 Docker 镜像进行部署。

## 2. 核心目标
1. 在该平台下完整实现现有功能模块。
2. 完成 Docker 镜像打包与部署。
3. 保证平台上的 Ansible 功能在 Docker 打包后仍可正常使用。
4. 镜像尽量精简，减少冗余脚本、无效脚本和无用资源。

## 3. 功能范围
- VDBench 可视化
- FIO 测试
- CosBench 配置生成
- 网段扫描
- 网络聚合配置
- 带宽测试
- 系统监控
- Ansible 管理平台
- 系统初始化

## 4. 重点要求
### 4.1 Docker 打包
- 前端和后端需可通过 Docker 一键启动。
- 容器启动后应自动提供前端页面与后端 API。
- 需保留必要的数据目录和配置目录。

### 4.2 Ansible 功能
- Docker 环境中需保持主机验证、命令执行、文件传输、Playbook 执行可用。
- 与 SSH 相关的依赖必须在镜像中可用。
- 不应因容器化导致现有功能退化。

### 4.3 镜像精简
- 删除无关脚本、临时文件和重复实现。
- 保留真正用于运行和构建的文件。
- 尽量缩小镜像体积，避免把开发缓存和构建产物一并打入运行镜像。

### 4.4 容器网络要求
- 容器部署建议使用 host 网络，确保平台内 SSH 能直接访问业务节点。
- 不建议使用默认 bridge 网络作为最终交付方案。

## 5. 交付标准
- 版本号升级为 2.0。
- 文档可作为后续开发、打包和验收依据。
- Docker 镜像可稳定启动并提供核心功能。
